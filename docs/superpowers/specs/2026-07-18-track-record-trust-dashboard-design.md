# Yansir 战绩页可信度总览设计

## 目标

将公开战绩页重设计为“策略可信度优先”的移动端页面，让新用户在五秒内理解：Yansir 使用真实信号、固定窗口和完整样本复盘策略表现。

选定视觉参考：`assets/track-record-trust-dashboard.png`。

## 范围

- 仅重设计公开 `track-record` 页面。
- 保留现有 API、缓存、筛选、分页、权限与登录回跳逻辑。
- 不改变 AI Claw、雷达、市场和账户页面。
- 不新增收益承诺、交易建议、时间序列图表或服务端尚未提供的指标。

## 核心用户问题

页面按顺序回答三个问题：

1. 这套策略当前表现如何？
2. 这个结论基于多少真实样本？
3. 我能否查看组成结论的具体记录和统计口径？

## 信息架构

### 1. 页面标题

- 标题：`历史战绩`。
- 副标题：`基于公开信号的真实执行记录，数据可验证。`
- 使用现有品牌色和图标系统；不引入新的插画资产。

### 2. 可信度总览

一个主表面承载三项已有摘要数据：

- 主指标：`1h 方向命中率`，使用 `directionalHitRate1h`。
- 样本上下文：在命中率旁展示 `totalSignals`，标签为 `公开信号样本`。当前 API 不提供 1h 命中分子，因此不得展示参考图中的 `(命中数 / 总数)`，也不得通过命中率反推该数字。
- 辅助指标：`公开信号`，使用 `totalSignals`。
- 辅助指标：`平均方向收益`，使用 `averageDirectionalReturn1h`。

可信度规则：

- 百分比必须与 `totalSignals` 样本规模同时出现，并明确这是公开信号总样本而非 1h 已完成样本数。
- 空值显示 `计算中`，不能显示虚构数字。
- `totalSignals = 0` 时，主区域保留结构并明确说明“暂无满足公开条件的样本”。

### 3. 筛选

- 保留币种输入与提交行为。
- 保留 `全部方向 / 看多 / 看空`。
- 移动端采用一行币种选择和一行方向分段控件；桌面端可并排。
- 筛选改变时继续沿用现有 URL 参数、缓存键和请求流程。

### 4. 最近公开信号

- 由现有公开战绩表转换为移动端分隔行列表。
- 每行展示币种、时间、方向、完成状态、1h 结果。
- 15m 结果在空间允许时作为次级字段显示。
- 4h、24h、MFE、MAE 继续遵守会员权限；匿名用户只看到简洁锁定提示。
- 点击或展开行为仅在已有数据与导航能力支持时实现，不新增不存在的详情路由。
- 桌面端仍可保留表格形态，但使用与移动端一致的信息优先级。

### 5. 统计口径

- 将当前三块方法论内容收敛为单个 `固定窗口，不挑样本` 折叠入口。
- 展开后展示：15m、1h、4h、24h 固定窗口、公开字段权限和服务端延迟约束。
- 默认折叠，避免方法说明压过战绩本身。

### 6. 导航

- 保持现有五项底部导航：市场、AI Claw、雷达、战绩、我的。
- `战绩` 保持激活态。
- 桌面端公共门户头部导航保持现状。

## 状态设计

### 加载

- 总览区域使用稳定骨架或 `正在加载公开战绩`，避免布局跳变。

### 空状态

- 标题：`暂无满足公开条件的样本`。
- 说明服务端延迟、公开历史范围和当前筛选条件。
- 提供清空币种或切换方向的现有操作，不补造示例信号。

### 不可用与缓存

- 有缓存时继续展示旧数据，并标出最后更新时间。
- 无缓存时显示重试按钮。
- 不以全屏错误覆盖底部导航。

### 权限锁定

- 4h、24h、MFE、MAE 使用低干扰锁定样式。
- 页面底部保留一次 `升级解锁完整战绩`，不在每一行重复促销按钮。

## 视觉规则

- 延续 Yansir 的白色、浅蓝、深海军蓝与克制绿色。
- 主指标使用绿色仅表达方向命中或正向结果，不表达收益保证。
- 优先使用留白、排版和分隔线；阴影保持轻微。
- 禁止卡片嵌套卡片、装饰性行情图、虚构币种图标和大面积渐变。
- 正文不小于 14px；交互目标最小高度 44px。

## 组件边界

- `TrackRecordHero`：标题与公开数据说明。
- `TrustSummary`：可信度主指标和样本上下文。
- `TrackRecordFilters`：币种与方向筛选。
- `TrackRecordList`：移动列表与桌面表格的共享数据映射。
- `MethodologyDisclosure`：统计口径折叠内容。
- `TrackRecordState`：加载、空、缓存过期与错误状态。

组件只接收格式化前的数据和回调；API 请求、缓存和 URL 同步仍由 `PublicTrackRecordView` 管理。

## 数据流

1. 页面从 URL 读取币种和方向。
2. 使用现有缓存键显示可用缓存。
3. 并行请求公开信号与战绩摘要。
4. 摘要进入 `TrustSummary`，信号进入 `TrackRecordList`。
5. 筛选变化触发原有请求、缓存和 URL 逻辑。
6. 加载更多继续使用现有分页字段。

## 验证标准

- 移动端首屏能看到标题、命中率、样本量、平均方向收益和筛选入口。
- 所有百分比均有样本上下文或 `计算中` 状态。
- 零样本时不出现虚构记录或虚构统计。
- 币种和方向筛选、重试、加载更多、升级回跳与底部导航继续可用。
- 匿名权限仍只公开 15m / 1h；受限字段不会泄露。
- 390px、768px、1180px 三个宽度无横向溢出；桌面表格仍可读。
- 键盘焦点、语义区域、状态播报和按钮可访问性保持或改善。

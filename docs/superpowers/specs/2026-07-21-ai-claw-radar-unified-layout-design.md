# AIClaw 与雷达统一布局设计

日期：2026-07-21

## 目标

以用户确认的组合视觉稿为唯一实现目标：

- 雷达采用策略信号操作台布局，优先展示确定性的策略事实、筛选与行内证据。
- AIClaw 采用“今日洞察 + 六个快捷入口 + 对话 + 信号上下文”布局。
- 两页使用同一套 Yansir 蓝色视觉系统、顶部状态、间距、图标和底部导航。

视觉目标：[`assets/ai-claw-radar-unified-layout.png`](assets/ai-claw-radar-unified-layout.png)

## 产品边界

- 雷达是实时策略信号和市场异动的事实入口。
- AIClaw 只解释、复核和归纳已有事实，不生成或覆盖策略方向、评分、价格、周期和时间。
- 现有 API、策略计算、权限和路由语义保持不变；本次只重组前端呈现和交互层级。
- 未登录状态保留现有登录限制，但登录提示应在目标布局内出现，不能跳转到无关页面或遮住整页结构。

## 统一视觉系统

- 页面底色：冷白或极浅蓝灰；功能区域优先使用同一页面平面和轻分隔线。
- 主色：Yansir cobalt `#2F6BFF`；机会和在线状态使用绿色；风险和看空使用红色；中性状态使用灰色。
- 圆角：主要面板 12px；标签和状态胶囊使用小圆角。
- 正文字号保持 14–16px；标题、事实和操作采用清晰的字重层级。
- 禁止卡片套卡片、过度阴影、促销横幅、钱包按钮、红色品牌化和大型吉祥物。
- 统一底部导航为：`数据 / AIClaw / 雷达 / 告警 / 我的`，当前项使用蓝色。
- 图标沿用项目现有图标体系；不可用 emoji、手绘 SVG 或 CSS 模拟图标。

## 雷达页面

### 第一屏结构

1. 顶部标题 `雷达`。
2. 右侧显示绿色监听状态和最近扫描时间。
3. 一级来源页签：`市场异动 / 策略信号 / 我的`，默认选中 `策略信号`。
4. 二级筛选：`全部 / 看多 / 看空 / 趋势突破 / 回调反弹 / 成交量异动 / 资金异动`，末尾保留高级筛选入口。
5. 主内容为单一时间线列表，不为每条记录创建独立卡片。

### 信号行

每行固定呈现：

- 触发时间
- 币种图标和交易对
- 周期
- 方向
- 策略评分
- 触发价格
- 展开/收起状态

机会使用绿色，风险或看空使用红色，等待/中性使用灰色。数据缺失时显示 `--`，不构造虚假数值。

### 行内展开

点击信号行在原位置展开：

- 最多两条简短证据，优先使用策略触发原因与已知指标。
- 操作固定为 `币种详情 / AIClaw 复核 / 加入观察`。
- `AIClaw 复核` 必须携带当前信号上下文。
- 再次点击当前行或点击收起图标关闭详情。

### 状态处理

- 加载：使用紧凑骨架行，不阻断顶部筛选。
- 空态：说明当前筛选下没有信号，并提供放宽筛选和查看扫描记录。
- 错误：显示最近可用数据和恢复提示，不把错误伪装成无信号。
- 权限受限：保留列表结构，仅对受限事实做延迟或遮罩说明。

## AIClaw 页面

### 第一屏结构

1. 顶部标题 `AIClaw`，旁边显示绿色 `在线` 或清晰的降级状态，右侧为帮助入口。
2. 今日洞察区标题为 `今天想先看什么？`，正文只保留一条市场概览，不展示未经数据支持的判断。
3. 六个两列快捷入口：`市场概览 / 资金流向 / 策略信号 / 热门代币 / 巨鲸动态 / 市场情绪`。
4. 一条简短助手欢迎消息。
5. 可选的信号上下文条，显示来源币种与事实摘要，并可进入或清除上下文。
6. 靠近底部的输入框和发送按钮；在移动端键盘出现时保持可用。

### 快捷入口行为

- 点击入口将对应问题填入输入框或立即发起已有安全查询。
- `策略信号` 默认带入最近选中的雷达信号；没有上下文时明确说明未选择信号。
- 对话响应沿用现有 API 和本地降级逻辑，并保留来源/模式提示。

### 登录与降级

- 未登录时仍展示页面结构和功能说明，在对话区域内显示登录引导。
- 大模型不可用时显示规则分析状态，不能显示为完全在线的大模型。
- 请求失败后保留用户问题，并展示可重试或本地上下文结果。

## 组件边界

避免继续扩大 `AppShell.tsx`，新增或提取以下视图组件：

- `RadarWorkspaceHeader`：标题、监听状态和来源页签。
- `RadarFilterBar`：二级筛选和高级筛选入口。
- `RadarSignalTimeline`：信号列表、时间线和分页状态。
- `RadarSignalRow`：事实行与行内展开。
- `AIClawOverview`：今日洞察和六个快捷入口。
- `AIClawConversation`：消息、信号上下文和加载/错误状态。
- `AIClawComposer`：输入、发送和移动端固定行为。

组件消费现有 API 形状和领域模型，不在展示组件中重新计算策略事实。

## 数据流

- 雷达继续从现有实时策略和市场异动数据构建 `LiveSignal`。
- 筛选只改变可见集合，不改写信号内容。
- 选择某条信号后，详情展开状态保存在雷达页面；跳转 AIClaw 时传递完整的已选信号上下文。
- AIClaw 快捷入口构造提示词，但最终响应仍来自现有 `/api/claw/chat`，失败时使用现有本地降级块。
- 底部导航继续使用现有 `view` 路由，不新增页面路由。

## 响应式行为

- 设计基准为 390 × 844。
- 320–430px 保持两列 AIClaw 快捷入口；文字过长时换行，不横向溢出。
- 雷达筛选栏可横向滚动，时间线主体不可横向滚动。
- 768px 以上页面保持移动端信息密度并限制内容宽度，不拉伸成稀疏桌面表格。
- 底部导航和输入框避开安全区域。

## 测试与验收

- 路由测试：`?view=radar` 和 `?view=claw` 正确渲染并保持底部导航激活态。
- 雷达测试：来源页签、二级筛选、行展开/收起、三项操作、空态、错误态和权限态。
- AIClaw 测试：六个快捷入口、信号上下文传递/清除、发送、加载、失败降级和未登录引导。
- 可访问性：所有按钮有可读名称；颜色不是唯一状态提示；触控目标至少 44px；动态状态使用合适的 `aria-live`。
- 视觉验收：在 390 × 844 下分别截取两页，与确认稿并排比较；检查间距、字体、圆角、分隔线、底部导航和溢出。
- 回归：现有雷达信号模型、策略数据、ValueClaw API 和其他导航页面不受影响。

## 完成标准

- 两页在视觉上明显属于同一产品。
- 雷达第一屏可快速扫描策略事实，展开后可以继续查看、复核或观察。
- AIClaw 第一屏能快速发起六类分析，并清楚显示当前信号上下文。
- AI 解释始终从属于策略事实。
- 移动端核心交互可用，且相关测试、类型检查和生产构建通过。
